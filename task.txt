Задача: разработать полноценный интернет-магазин по продаже LEGO на Django (Python) с нуля. Включает регистрацию и авторизацию пользователей (без подтверждения по почте, просто логин/пароль), каталог товаров с категориями, карточки товаров, корзину. Администратор может управлять категориями и товарами через встроенную админ-панель.

Стек: Python 3.10+, Django 4.2+, SQLite (или PostgreSQL), Bootstrap 5 для стилизации (если нужно), стандартные средства Django для аутентификации и сессий.

Основные требования:
Регистрация и авторизация

Форма регистрации (имя, email, пароль, подтверждение пароля). Email не проверять – просто сохранять.

Форма входа (логин/пароль).

Выход из аккаунта.

Доступ к корзине и оформлению заказа только для авторизованных пользователей (но просмотр каталога – без регистрации).

Использовать встроенные django.contrib.auth.views или написать свои вьюхи.

Модель категории

Поля: название (уникальное), slug (генерируется автоматически), описание (опционально), порядок сортировки, активна.

Возможность древовидной структуры (опционально: необязательно для первой версии, но если сделать – плюс).

В админ-панели: добавление, редактирование, удаление, поиск по названию.

Модель товара (набор LEGO)

Поля: название, категория (ForeignKey), артикул (уникальный), цена (decimal), старая цена (опционально), количество на складе (integer), краткое описание, полное описание, изображение (ImageField), дата добавления, дата обновления, активен (булево).

Дополнительные поля по желанию: возрастная группа, количество деталей, серия.

Slug для красивых URL.

В админ-панели: list_display, list_filter по категории и активности, search_fields по названию и артикулу.

Каталог товаров

Главная страница /catalog/ со списком всех активных товаров.

Пагинация (например, 12 товаров на страницу).

Фильтр по категории (выпадающий список или сайдбар).

Сортировка: по цене (возрастание/убывание), по дате добавления (новинки), по названию.

Поиск по названию и краткому описанию.

Каждый товар в виде карточки: изображение, название, цена, кнопка "В корзину" (добавляет товар в корзину через AJAX или POST-запрос).

Карточка товара

URL: /product/<slug>/.

Полная информация: название, цена, наличие, описание, изображение, характеристики.

Кнопка "Добавить в корзину" с выбором количества (1 шт. по умолчанию).

Блок "Похожие товары" (из той же категории, до 4 штук).

Хлебные крошки: Главная → Категория → Товар.

Корзина

Реализовать с использованием сессий (для неавторизованных) и/или привязки к пользователю.

Возможность добавить товар с количеством, изменить количество, удалить товар из корзины.

Отображение корзины на отдельной странице /cart/ с таблицей: товар, цена, количество, общая сумма за позицию, общая сумма корзины.

Кнопка "Очистить корзину".

Кнопка "Перейти к оформлению" (страницу оформления пока не делаем, просто заглушка).

В шапке сайта отображать количество товаров в корзине и общую сумму.

Админ-панель

Зарегистрировать модели Category и Product с настройками:

prepopulated_fields: {'slug': ('name',)}.

В Product добавить actions для массового включения/отключения активности.

Права: только суперпользователь или staff могут управлять товарами/категориями.

Шаблоны и статика

Базовый шаблон base.html с шапкой (логотип, поиск, корзина, ссылки на вход/регистрацию/личный кабинет).

Для стилизации использовать Bootstrap 5 (подключить CDN).

Настроить MEDIA_URL и MEDIA_ROOT для загрузки изображений.

Что нужно сгенерировать:
Полные файлы проекта Django (settings.py, urls.py и т.д.).

Модели (models.py) с описанными полями.

Админ-классы (admin.py).

Вьюхи (классы или функции) для каталога, карточки товара, корзины, регистрации/авторизации.

Формы (регистрация, добавление в корзину, изменение количества).

URL-маршруты для всех разделов.

Шаблоны (catalog/product_list.html, product_detail.html, cart/cart_detail.html, registration/register.html, login.html).

Настройки сессий для корзины.

Команду для загрузки демо-категорий и товаров (фикстуры или скрипт).

Учесть:
Все тексты в коде на русском или английском – на ваше усмотрение, но verbose_name полей должны быть на русском.

Для изображений товаров в админке предусмотреть превью (опционально: использовать easy_thumbnails или просто отображать ссылку).

Корзина должна работать и для авторизованных, и для неавторизованных пользователей. При логине корзина из сессии не переносится (можно и переносить, но для простоты пока не требуется).

Обработка ошибок: если товара нет в наличии, нельзя добавить больше, чем на складе (сделать проверку).

Добавить сообщения (django.contrib.messages) для уведомлений об успешном добавлении в корзину, ошибках и т.д.

Финальный результат:
Полностью рабочий Django-проект, который после выполнения миграций и создания суперпользователя позволяет:

Зарегистрировать нового пользователя.

Войти и выйти.

Добавлять категории и товары через админку.

Просматривать каталог с фильтрацией и поиском.

Открыть карточку товара.

Добавлять товары в корзину, управлять корзиной.

Видеть обновляемую сумму и количество товаров в шапке.
